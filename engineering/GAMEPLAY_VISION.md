# Gameplay Vision

Use this as a decision guardrail for gameplay work. It is not a feature roadmap.

## Combat

- Fights should be readable, fair, and fast enough for mobile sessions.
- Players should understand why they won or lost.
- Basic attacks matter; skills add timing, identity, and moments without deciding every fight instantly.
- Defensive actions should have visible value when supported.

## Classes

- Warrior: durable, stable pressure.
- Rogue: burst and mobility, fragile.
- Mage: skill/control/range fantasy, fragile.
- Class identity should be visible in skills, feedback, and UI labels.

## Mobile First

- Controls must be comfortable on small screens.
- Important state must be glanceable.
- Avoid long explanations during active play.

## PvP And Arena

- PvP should feel competitive, understandable, and responsive.
- Arena entry, objectives, combat state, and results should be obvious.
- Do not hide decisive information behind unclear UI.

## Boss Raid

- Boss Raid should communicate boss state, player contribution, danger, and reward/progression clearly.
- Skill use and hit feedback should be easy to read.

## Progression

- Progression should show growth and next goals without implying pay-to-win.
- Rewards should support retention through clarity, not confusion.
